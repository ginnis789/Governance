5.2 Configure Transaction Stringing

The child template, which is the XXExtend, will receive the Return Value from the Parent Template; the XXMaterials_JDE template.

To configure Transaction Stringing:

  1. Click Loops for the XXExtendToPlant.
  2. Locate RMMG1-MATNR in the Dependent pane.

NOTE:This field will use the Return Value from the Parent Template.

  1. Click Edit
  2. Select [None] from Mapping Value list box.
  3. Click the Save button.
  4. Click Vertical View for RMMG1-MATNR.
  5. Click the Edit button.

NOTE:A field cannot contain both a Mapping Value and a Return Value. It either has to have a value supplied from the view, or a Return Value supplied from the Parent Template.

  1. Select XXMaterials_JDE from Return Value Template ID list box.

NOTE:The Return Value Template ID is the Template ID of a lower priority that will be executed before the current Template.

  1. Select Return Value 1 from Return Value list box.
  2. Click the Save button.

Transaction Stringing

NOTE:The Return Value column contains the data to pass to the current Template Field. SAP returns up to 4 return values for each posting and the return values differ for each transaction code. The return values make up the dynamic part of the message that is displayed to the user in the status bar after saving the record.

For example:

 

MM01: Message displayed “Material 101 created successfully”

Return Value 1: 101 = Material Number

FB50: Message displayed “Document 123235 was posted in company code 0001”

Return Value 1: 123235 = Document Number

Return Value 2: 0001 = Company Code

NOTE:In order to determine the Return Values, it is recommended that the transaction used by the Template be executed manually in SAP and the status bar monitored in order to determine which return values are passed back via SAP.

  1. Close the Vertical View.
  2. Verify Using Return Value (RV) field is checked.
