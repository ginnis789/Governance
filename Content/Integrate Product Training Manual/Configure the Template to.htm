7.1 Configure the Template to Loop

Looping is a posting mechanism that processes multiple child keys for one parent key. It allows for multiple headers with infinite details to be posted. Looping can be enabled during template creation or after recording.

NOTE:When a template is added to a process, one loop is automatically created, regardless if looping is enabled. Technically, every template has at least a loop of one header.

When a template that uses loops is added to a process, each loop in the template becomes a process template loop. Looping is necessary when data in the same field at the detail level should be updated for multiple header records. For example, to enter data for the Export Legal Control for a Material, a template would need to generate one transaction with multiple screens to update the export control class for each country.

Looping is not necessary when a change in data is made at the header level as is the case when changing a Material Description.

An example of when looping is necessary is when an Accounting Document is created.

To create a loop, create a GL Account with multiple Components to update. Depending on the GL Account, it can have several components or just a single component.
