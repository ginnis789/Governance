Exercise 10 - Post Using a BAPI

Integrate supports the execution of any RFC enabled Function Module in SAP, including the execution of RFCs and the posting of data using BAPIs.

NOTE:Before creating a BAPI/RFC template, the RFC functions must be extracted from SAP using the SAP Connections page.

A process based on a BAPI template will post data bi-directionally. Data is uploaded to be posted into SAP and downloaded into tables that Integrate automatically creates. Since processes based on BAPI templates often have return values, download registrations are often required for BAPI templates.

BAPI template types can use transaction stringing to pass return values to other templates assigned to a process.

Integrate is an automation tool that allows experienced SAP users to leverage data loading capabilities provided by SAP. Knowledge of how to use SAP loading technologies such as BAPIs is required to use Integrate, and the documentation and technical support included does not cover specific usage details or how these technologies work.

To post data using a BAPI:

  • Configure a BAPI Template.
  • Activate the Template.
  • Add the Template to a Process.
  • Configure Process Template Loops for a BAPI Template.
  • View and Configure relationships.
  • View and Edit field mappings.
  • Activate the Process.
  • Verify the Tables and Views were created in SQL.
  • Use a Process Post to post data.
  • Verify the Data was posted in SAP.
