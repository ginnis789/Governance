Exercise 1 - Post Using BDC Recording

Post data to SAP using a Batch Data Communication (BDC) script recording, using either Integrate or an imported SHDB transaction in SAP.

Integrate supports multiple system connections.

A template defines how data is posted into external systems. In this example, we are working through a BDC posting, including which screens and fields will be processed and whether the process contains a single loop or multiple loops, which will be covered in a later section of the training. Templates are not tied to data; they are an independent guide for posting that can be assigned to one or more processes.

NOTE:A BDC Script template can be created by copying an existing template. Templates can then be merged with another template.

Once a template is created, it is added to a process. A process, a series of posting steps, defines how Integrate takes data from the platform and transfers it to the business systems. Each posting step is a template that is added to the process.

Once assigned to a process, the BDC Script template can be further configured to include transaction stringing, template process loops tied to different views that contain the data to post to SAP, and field mappings to a specified field in a data model or a fixed value. Integrate can also map many fields in the template with columns in an assigned view automatically.

After the process is configured and activated, use a posting process to send the data to SAP.

BDC script posting process:

  • Create a Category to organize templates and processes
  • Record a BDC script or Import a BDC script from a file
  • Configure conditional BDC screens (optional)
  • Manage BDC screen fields (optional)
  • Configure template loops
  • Activate the template
  • Add the template to a process
  • Create and configure process template loops
  • Configure field mappings
  • Configure relationships
  • Activate the process
