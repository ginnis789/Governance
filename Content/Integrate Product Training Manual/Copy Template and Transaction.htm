Exercise 5 - Copy Template and Transaction Stringing

If the Process has multiple Templates assigned, Templates with a greater priority than the initial Template can use Return Values from previous Templates in their posting. The process is called Transaction Stringing. Transaction Stringing can utilize process information from one template (transaction) in a second template (transaction), or it can process two like transactions within one upload. An example of the former is one template creating a material (MM01), then a second template creating a corresponding info record (ME11). Because the material number is not known at the time the second data source is created, it can be passed in from the first data source after it has processed. In order to take advantage of the Transaction Stringing, the Vertical View of the Process Template Loop Field Mapping allows for the mapping of return values from a previous template to a Template Field of the current template.

NOTE:Only multiple template processes based on a BDC Script, GUI Script or BAPI/RFC template types can use Transaction Stringing. The process must use the synchronous post execution method.

Create a process with two templates. The first creates a set of finished materials, and then extends newly created materials to a plant. Use the MM01 Integrate Materials template for the first template. Then create a new template to extend the material to a new plant by copying an existing template.

The following steps are required:

  • Create a Template by copying one from another Category
  • Add a Process to TrainingXX Category.
  • Add a Second Template to the Process.
  • Configure Return Value Parameters.
  • Post the changes.
